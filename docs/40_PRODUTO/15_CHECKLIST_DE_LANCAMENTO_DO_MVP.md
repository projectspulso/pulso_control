# Checklist de Lancamento do MVP

## Produto

- [ ] O posicionamento oficial esta fechado
- [ ] O publico-alvo esta claro
- [ ] A tese do MVP cabe em 1 frase
- [ ] As nao promessas estao documentadas

## Tecnologia

- [ ] O app sobe localmente
- [ ] O banco tem lote minimo definido
- [ ] O n8n esta alinhado com o app
- [ ] Os webhooks principais respondem
- [ ] Nao restam hardcodes sensiveis em scripts criticos

## Operacao

- [ ] Existe 1 canal foco
- [ ] Existe 1 formato principal
- [ ] A equipe sabe aprovar ideias e roteiros
- [ ] O pipeline de producao esta utilizavel
- [ ] A publicacao assistida esta compreendida pela equipe

## Validacao

- [ ] O fluxo completo rodou ao menos 2 vezes
- [ ] Existe 1 lote de 5 a 10 posts para avaliar
- [ ] A equipe sabe explicar o que funcionou e o que falhou
- [ ] O produto reduziu friccao operacional de verdade

## Decisao

- [ ] GO
- [ ] AJUSTAR
- [ ] KILL
